# MediClear — Database Schema (Supabase / Postgres)

## `users`
Managed by Supabase Auth automatically — no need to create this manually.
| Column | Type | Notes |
|---|---|---|
| id | uuid | primary key, auto-generated by Supabase Auth |
| email | text | user's email |
| created_at | timestamp | auto-set on signup |

## `reports`
```sql
create table reports (
  id uuid primary key default gen_random_uuid(),
  user_id uuid references auth.users(id) not null,
  uploaded_at timestamp default now(),
  file_type text not null,        -- 'pdf' or 'image'
  raw_text text,                  -- extracted text before AI analysis
  status text default 'processing' -- 'processing' | 'complete' | 'failed'
);
```

## `test_values`
```sql
create table test_values (
  id uuid primary key default gen_random_uuid(),
  report_id uuid references reports(id) not null,
  test_name text not null,
  value text not null,
  reference_range text,
  status text not null,           -- 'Normal' | 'Monitor' | 'Consult Doctor Soon'
  explanation text not null,
  specialist_suggestion text
);
```

## `agent_logs`
Stores each step of the agentic reasoning chain for a report — used for demoing
the "agentic AI use case" during evaluation.
```sql
create table agent_logs (
  id uuid primary key default gen_random_uuid(),
  report_id uuid references reports(id) not null,
  step_name text not null,        -- 'extract' | 'self_assess' | 'deep_reasoning' | 'finalize'
  reasoning_output jsonb,         -- the raw output of that step
  duration_ms integer,            -- for latency stats (Milestone 5)
  created_at timestamp default now()
);
```

## Relationships
- One `user` → many `reports`
- One `report` → many `test_values`
- One `report` → many `agent_logs` (one row per reasoning step)
